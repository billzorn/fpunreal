about Titanic

Titanic is an implementation of IEEE 754 floating point arithmetic in terms of real numbers. It is suitable as a reference; though it is much slower than numerical libraries such as GMP and NumPy, Titanic has a very simple implementation and is naturally immune to quantization error due to floating point rounding.

Typical numerical libraries use a binary implementation of floating point for maximum performance. However, the IEEE 754 standard is specified in terms of real numbers. Ensuring that the behavior of a discrete program lines up with this continuous specification can be difficult.

One way of thinking about a specific IEEE 754 floating point format with some precision is as a map from bit patterns to a set of real numbers. But it is also a map from a set of intervals that cover the real line, or "rounding envelopes," to the single representative number chosen for each interval.

Given a number, Titanic can efficiently find the interval in some floating point format that contains that number. Given a computation, Titanic can find the interval that contains the real value of the result. Because it uses real numbers, Titanic can implement the IEEE 754 specification directly, without having to reason about the behavior of a binary representation.

Titanic is effectively an "infinite precision" floating point implementation: given any computation and any floating point format, it will either produce a correctly rounded answer or exhaust its available computing resources and report that it has failed. It is guaranteed to float correctly.

floating point constants

Titanic will tell you everything you could possibly want to know about a number in an IEEE 754 floating point format of any precision, or about how any real number would be represented in such a format.

providing input to Titanic

Titanic needs two kinds of input: some representation of a number, and the precision of the IEEE 754 floating point format that number will be represented in.
